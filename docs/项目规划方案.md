# 家庭中心设备项目规划方案

## 项目概述

基于 RK3588 开发一款家庭中心设备，通过接入摄像头及其他 IoT 设备，利用 MediaPipe 实现目标检测和行为分析，为家庭提供智能安防、老人儿童看护、语音交互等功能。

---

## 一、硬件选型方案

### 1.1 核心处理器 - RK3588

| 规格 | 参数 |
|------|------|
| CPU | 4×Cortex-A76 @ 2.4GHz + 4×Cortex-A55 @ 1.8GHz |
| NPU | 6TOPS (INT8) |
| GPU | Mali-G610 MP4 |
| 视频解码 | 8K@60fps / 4K@120fps |
| 视频编码 | 8K@30fps / 4K@60fps |
| 内存 | 支持 LPDDR5/LPDDR4X |
| 接口 | PCIe 3.0, USB 3.0, MIPI-CSI, HDMI 2.1, GMAC |

**选择理由**：6TOPS NPU 足以运行 MediaPipe 实时推理，8K 视频能力支持多路高清摄像头接入。

### 1.2 硬件模块选型

| 模块 | 选型 | 说明 |
|------|------|------|
| 核心板 |  RK3588 SOM | 集成处理器、PMU、内存 |
| 底板 | 自研家庭网关底板 | 扩展接口 |
| 摄像头接口 | MIPI-CSI + USB | 支持多路摄像头 |
| 存储 | eMMC 5.1 + TF卡 | 系统+录像存储 |
| 网络 | 千兆网口 + Wi-Fi 6 | 有线/无线双路 |
| 音频 | I2S 麦克风阵列 | 语音唤醒 + 双向通话 |
| 显示 | HDMI 2.1 输出 | 可选连电视/显示器 |
| 电源 | 12V/2A DC | 适配器供电 |

### 1.3 设备接入能力

- **视频输入**：4路 MIPI-CSI + 2路 USB 摄像头
- **IoT 协议**：Zigbee 3.0 / Wi-Fi / 蓝牙 5.0 / Matter
- **有线接口**：RS485（智能家居总线）

## 二、软件架构设计

### 2.1 系统架构图

```
┌─────────────────────────────────────────────────────────┐
│                    应用层 (App Layer)                    │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────────┐   │
│  │安防监控│ │人脸识别│ │行为分析│ │语音助手    │   │
│  └─────────┘ └─────────┘ └─────────┘ └─────────────┘   │
├─────────────────────────────────────────────────────────┤
│                   框架层 (Framework)                     │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐    │
│  │  MediaPipe   │ │  HomeKit     │ │  MQTT Broker │    │
│  │  (AI 推理)   │ │  (设备管理)   │ │  (消息总线)   │    │
│  └──────────────┘ └──────────────┘ └──────────────┘    │
├─────────────────────────────────────────────────────────┤
│                    驱动层 (Driver)                       │
│  Camera HAL │ NPU Driver │ GPU Driver │ Wi-Fi/BT Driver│
├─────────────────────────────────────────────────────────┤
│                  系统层 (OS/RTOS)                         │
│        Linux 5.10 (主系统) + RTOS (实时任务)            │
└─────────────────────────────────────────────────────────┘
```

### 2.2 关键软件组件

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 操作系统 | Debian/Ubuntu ARM64 | 成熟稳定，生态丰富 |
| 容器引擎 | Docker | 应用隔离部署 |
| AI 框架 | MediaPipe | Google 出品，高效推理 |
| 设备管理 | Home Assistant | 开源智能家居平台 |
| 消息中间件 | Mosquitto MQTT | 轻量级 IoT 消息总线 |
| 视频存储 | SQLite + 文件存储 | 本地录像存储 |
| Web 服务 | Flask/FastAPI | 移动端/Web 控制界面 |

### 2.3 MediaPipe 集成方案

```
MediaPipe 任务清单：
├── Face Detection     → 人脸检测/识别
├── Face Mesh         → 表情分析
├── Pose              → 人体姿态检测（跌倒检测）
├── Hand Tracking     → 手势控制
├── Object Detection  → 物品识别（包裹检测）
└── Gesture Recognizer→ 日常行为识别
```

**性能优化策略**：
- NPU 加速：使用 RKNN 工具链转换模型
- 批处理：多路视频流并行推理
- 异步处理：推理与视频采集解耦

## 三、功能规划

### 3.1 核心功能模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **智能安防** | 人形检测+入侵报警+手机推送 | P0 |
| **人脸识别** | 家庭成员识别+陌生人检测 | P0 |
| **跌倒检测** | 老人摔倒自动报警 | P1 |
| **区域入侵** | 禁区闯入检测 | P1 |
| **语音交互** | 语音唤醒+语音控制 | P1 |
| **远程监控** | 手机随时查看实时画面 | P0 |
| **云端存储** | 报警录像云端备份 | P2 |
| **IoT 控制** | 联动灯光/门锁/传感器 | P2 |

### 3.2 用户场景

1. **离家模式**：离家自动布防，异常入侵即时告警
2. **在家模式**：看护老人儿童，异常行为及时通知
3. **看护模式**：婴儿/老人活动监控，紧急情况一键求助

### 3.3 报警通知

- App 推送（极光/FCM）
- 短信通知（可选）
- 本地声光报警

### 4.3 关键风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| NPU 模型转换 | 影响 AI 性能 | 预留 GPU/CPU 回退方案 |
| 多路视频延迟 | 实时性差 | 优化推理 pipeline |
| 发热控制 | 稳定性问题 | 优化散热设计 |
| 隐私安全 | 用户信任 | 端侧处理+加密传输 |

## 五、技术指标

| 指标 | 目标值 |
|------|--------|
| 人脸检测延迟 | < 100ms |
| 跌倒检测延迟 | < 500ms |
