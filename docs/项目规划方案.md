# 家庭中心设备项目规划方案

## 项目概述

基于 RK3588 开发一款家庭中心设备，通过接入摄像头及其他 IoT 设备，利用 MediaPipe 实现目标检测和行为分析，为家庭提供智能安防、老人儿童看护、语音交互等功能。

---

## 一、硬件选型方案

### 1.1 核心处理器 - RK3588

| 规格 | 参数 |
|------|------|
| CPU | 4×Cortex-A76 @ 2.4GHz + 4×Cortex-A55 @ 1.8GHz |
| NPU | 6TOPS (INT8) |
| GPU | Mali-G610 MP4 |
| 视频解码 | 8K@60fps / 4K@120fps |
| 视频编码 | 8K@30fps / 4K@60fps |
| 内存 | 支持 LPDDR5/LPDDR4X |
| 接口 | PCIe 3.0, USB 3.0, MIPI-CSI, HDMI 2.1, GMAC |

**选择理由**：6TOPS NPU 足以运行 MediaPipe 实时推理，8K 视频能力支持多路高清摄像头接入。

### 1.2 硬件模块选型

| 模块 | 选型 | 说明 |
|------|------|------|
| 核心板 |  RK3588 SOM | 集成处理器、PMU、内存 |
| 底板 | 自研家庭网关底板 | 扩展接口 |
| 摄像头接口 | MIPI-CSI + USB | 支持多路摄像头 |
| 存储 | eMMC 5.1 + TF卡 | 系统+录像存储 |
| 网络 | 千兆网口 + Wi-Fi 6 | 有线/无线双路 |
| 音频 | I2S 麦克风阵列 | 语音唤醒 + 双向通话 |
| 显示 | HDMI 2.1 输出 | 可选连电视/显示器 |
| 电源 | 12V/2A DC | 适配器供电 |

### 1.3 设备接入能力

- **视频输入**：4路 MIPI-CSI + 2路 USB 摄像头
- **IoT 协议**：Zigbee 3.0 / Wi-Fi / 蓝牙 5.0 / Matter
- **有线接口**：RS485（智能家居总线）

### 2.3 AI 框架与模型选型

#### 2.3.1 混合 AI 架构

由于新需求要求更全面的行为分析和目标检测，我们将采用 **多模型协同** 架构：

| AI 任务 | 推荐模型 | 框架 | 说明 |
|---------|----------|------|------|
| 人脸检测/识别 | MediaPipe Face Detection + Face Mesh | MediaPipe | 成熟稳定，低延迟 |
| **人体行为识别** | **YOLO-Pose +行为分类模型** | **ONNX Runtime** | **支持多种姿态和行为** |
| **目标检测** | **YOLO v8/v11** | **ONNX Runtime** | **80+ 类物体检测** |
| 手势控制 | MediaPipe Hands | MediaPipe | 精细手势识别 |
| 物品识别 | MediaPipe Object Detection | MediaPipe | 基础物品识别 |
| 跌倒检测 | 自研时序模型 + Pose 特征 | ONNX Runtime | 基于姿态序列分析 |

#### 2.3.2 YOLO 模型优势

```
YOLO (You Only Look Once) 系列：
├── YOLO v8  - 当前最强通用目标检测模型
│   ├── YOLOv8n (nano) - 极轻量，适合边缘设备
│   ├── YOLOv8s (small) - 平衡精度与速度
│   └── YOLOv8m (medium) - 高精度
├── YOLO-Pose - 实时人体姿态估计
│   ├── 17 个关键点检测
│   └── 行为分类（站立/走路/跌倒/坐下/摔倒等）
└── 模型优化
    ├── 模型量化 (FP16/INT8)
    └── NPU 加速 (RKNN)
```

#### 2.3.3 行为识别能力扩展

**新增支持的行为分析**：

| 行为类别 | 具体动作 | 应用场景 |
|----------|----------|----------|
| 跌倒类 | 摔倒、跌倒、滚落 | 老人看护 |
| 活动类 | 站立、行走、跑步、坐下、起立 | 活动监控 |
| 危险类 | 挥手求救、长时间静止、进入禁区 | 安防报警 |
| 日常类 | 吃饭、喝水、打电话、使用物品 | 生活分析 |
| 异常类 | 陌生人闯入、遗留物品、物品搬动 | 安防监控 |

#### 2.3.4 目标检测能力扩展

**新增支持的目标类别**（基于 COCO 80 类 + 自定义类）：

| 类别 | 检测目标 |
|------|----------|
| 人物 | 人、婴儿、小孩、老人 |
| 物品 | 包裹、箱子、书包、手机、钥匙 |
| 宠物 | 猫、狗 |
| 车辆 | 汽车、自行车、摩托车 |
| 家具 | 椅子、桌子、床、沙发 |
| 自定义 | 燃气灶、刀具、药品等特定目标 |

#### 2.3.5 部署架构

```
┌─────────────────────────────────────────────────────────────┐
│                    AI 推理层 (NPU 加速)                     │
│  ┌─────────────────┐  ┌─────────────────┐                  │
│  │  MediaPipe      │  │  ONNX Runtime   │                  │
│  │  (人脸/手势/姿态)│  │  (YOLO 检测)    │                  │
│  └─────────────────┘  └─────────────────┘                  │
├─────────────────────────────────────────────────────────────┤
│                    模型转换层                               │
│  ┌─────────────────┐  ┌─────────────────┐                  │
│  │  MediaPipe      │  │  RKNN Toolkit   │                  │
│  │  模型库          │  │  (YOLO→NPU)     │                  │
│  └─────────────────┘  └─────────────────┘                  │
├─────────────────────────────────────────────────────────────┤
│                    推理调度层                               │
│  ┌─────────────────────────────────────────────────┐        │
│  │           多模型并行推理调度器                    │        │
│  │  • 视频流管理  • 推理队列  • 结果融合            │        │
│  └─────────────────────────────────────────────────┘        │
└─────────────────────────────────────────────────────────────┘
```

**性能优化策略**：
- NPU 加速：使用 RKNN 工具链转换 YOLO 模型到 NPU 格式
- 模型量化：INT8 量化，精度损失 < 2%
- 批处理：多路视频流并行推理
- 异步处理：推理与视频采集解耦
- 内存优化：模型分片加载，按需加载

## 三、功能规划

### 3.1 核心功能模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **智能安防** | 人形检测+入侵报警+手机推送 | P0 |
| **人脸识别** | 家庭成员识别+陌生人检测 | P0 |
| **跌倒检测** | 老人摔倒自动报警（多姿态分析） | P0 |
| **区域入侵** | 禁区闯入检测 | P0 |
| **全面行为分析** | 站立/行走/跌倒/坐下/挥手等行为识别 | P0 |
| **全面目标检测** | 80+类物体检测（包裹/宠物/车辆/家具等） | P0 |
| **语音交互** | 语音唤醒+语音控制 | P1 |
| **远程监控** | 手机随时查看实时画面 | P0 |
| **云端存储** | 报警录像云端备份 | P2 |
| **IoT 控制** | 联动灯光/门锁/传感器 | P2 |

### 3.2 用户场景

1. **离家模式**：离家自动布防，异常入侵即时告警，遗留物品检测
2. **在家模式**：看护老人儿童，全面的行为分析（跌倒/活动/危险行为）
3. **看护模式**：婴儿/老人活动监控，紧急情况一键求助，宠物活动追踪

### 3.3 报警通知

- App 推送（极光/FCM）
- 短信通知（可选）

## 六、总结

本方案基于 RK3588 的强劲 NPU 算力，结合 **MediaPipe + YOLO** 的混合 AI 架构，构建一款功能完整的家庭中心设备。

**核心升级**：
- ✅ 引入 YOLO v8/v11 实现 80+ 类全面目标检测
- ✅ 引入 YOLO-Pose 实现更精确的人体姿态估计
- ✅ 新增 10+ 种行为识别（跌倒/活动/危险行为）
- ✅ 采用 ONNX Runtime + RKNN NPU 加速

硬件选型兼顾性能与成本，软件架构采用模块化设计，支持灵活扩展。

**下一步行动**：
1. 确认硬件选型清单
2. 启动硬件原理图设计
3. 搭建开发环境
4. **YOLO 模型选型测试**（推荐 YOLOv8n/YOLOv8s）
