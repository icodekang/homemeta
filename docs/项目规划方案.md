# 家庭中心设备项目规划方案

## 项目概述

基于 RK3588 开发一款家庭中心设备，通过接入 6 路摄像头及其他 IoT 设备，利用 **纯 YOLO** 实现目标检测和行为分析（不适用 MediaPipe），为家庭提供智能安防、老人儿童看护、语音交互等功能。

---

## 一、硬件选型方案

### 1.1 核心处理器 - RK3588

| 规格 | 参数 |
|------|------|
| CPU | 4×Cortex-A76 @ 2.4GHz + 4×Cortex-A55 @ 1.8GHz |
| NPU | 6TOPS (INT8) |
| GPU | Mali-G610 MP4 |
| 视频解码 | 8K@60fps / 4K@120fps |
| 视频编码 | 8K@30fps / 4K@60fps |
| 内存 | 支持 LPDDR5/LPDDR4X |
| 接口 | PCIe 3.0, USB 3.0, MIPI-CSI, HDMI 2.1, GMAC |

**选择理由**：6TOPS NPU 足以运行 YOLO 实时推理，8K 视频能力支持多路高清摄像头接入。

### 1.2 硬件模块选型

| 模块 | 选型 | 说明 |
|------|------|------|
| 核心板 |  RK3588 SOM | 集成处理器、PMU、内存 |
| 底板 | 自研家庭网关底板 | 扩展接口 |
| 摄像头接口 | MIPI-CSI + USB | 支持多路摄像头 |
| 存储 | eMMC 5.1 + TF卡 | 系统+录像存储 |
| 网络 | 千兆网口 + Wi-Fi 6 | 有线/无线双路 |
| 音频 | I2S 麦克风阵列 | 语音唤醒 + 双向通话 |
| 显示 | HDMI 2.1 输出 | 可选连电视/显示器 |
| 电源 | 12V/2A DC | 适配器供电 |

### 1.3 设备接入能力

- **视频输入**：4路 MIPI-CSI + 2路 USB 摄像头（共 6 路）
- **IoT 协议**：Zigbee 3.0 / Wi-Fi / 蓝牙 5.0 / Matter
- **有线接口**：RS485（智能家居总线）

### 2.3 AI 框架与模型选型

#### 2.3.1 纯 YOLO 架构

**新方案：只使用 YOLO，不使用 MediaPipe**

| AI 任务 | YOLO 模型 | 说明 |
|---------|-----------|------|
| **目标检测** | YOLOv8s / YOLOv11 | 80+ 类物体检测 (COCO) |
| **人体姿态估计** | YOLO-Pose | 17 个关键点检测 |
| **行为识别** | 基于 Pose 的时序分类 | 站立/走路/跌倒/坐下等 |
| **人脸检测** | YOLOv8-Face | YOLO 系列人脸检测模型 |

#### 2.3.2 YOLO 模型优势

```
YOLO (You Only Look Once) 系列：
├── YOLOv8  - 当前最强通用目标检测模型
│   ├── YOLOv8n (nano) - 极轻量，适合边缘设备
│   ├── YOLOv8s (small) - 平衡精度与速度 ✓推荐
│   └── YOLOv8m (medium) - 高精度
├── YOLO-Pose - 实时人体姿态估计
│   ├── 17 个关键点检测
│   └── 行为分类（站立/走路/跌倒/坐下/摔倒等）
└── YOLO-Face - 人脸检测
    ├── 无需额外框架
    └── 统一推理引擎
```

#### 2.3.3 行为识别方案（基于 YOLO-Pose）

**不使用 MediaPipe，改用 YOLO-Pose + 时序分类**：

```
YOLO-Pose 检测 → 关键点坐标 → 特征提取 → 时序分类模型 → 行为类别
```

| 行为类别 | 具体动作 | 检测方式 |
|----------|----------|----------|
| 跌倒类 | 摔倒、跌倒、滚落 | 姿态角度 + 运动速度 |
| 活动类 | 站立、行走、跑步、坐下、起立 | 关键点位置序列 |
| 危险类 | 挥手求救、长时间静止、进入禁区 | 区域检测 + 姿态 |
| 异常类 | 陌生人闯入、遗留物品、物品搬动 | 目标检测 + 区域 |

#### 2.3.4 目标检测能力

**基于 COCO 80 类 + 自定义类**：

| 类别 | 检测目标 |
|------|----------|
| 人物 | 人、婴儿、小孩、老人 |
| 物品 | 包裹、箱子、书包、手机、钥匙 |
| 宠物 | 猫、狗 |
| 车辆 | 汽车、自行车、摩托车 |
| 家具 | 椅子、桌子、床、沙发 |
| 自定义 | 燃气灶、刀具、药品等特定目标 |

## 三、性能可行性分析（关键）

### 3.1 6 路 1080P@1fps 可行性分析

#### 3.1.1 理论计算

| 参数 | 数值 |
|------|------|
| 视频路数 | 6 路 |
| 分辨率 | 1920×1080 (1080P) |
| 帧率 | 1 fps/路 |
| 总处理帧率 | 6 fps |
| 单帧处理时间 | ≤ 166ms (保证实时) |

#### 3.1.2 RK3588 NPU 性能

| 模型 | 输入尺寸 | INT8 推理时间 | 适用场景 |
|------|----------|---------------|----------|
| YOLOv8s | 640×640 | ~15ms | 目标检测 |
| YOLO-Pose | 640×640 | ~20ms | 姿态估计 |
| YOLO-Face | 640×640 | ~10ms | 人脸检测 |

#### 3.1.3 6 路并发处理分析

```
方案：串行 + 异步调度

每帧处理流程（1秒周期）：
├── T0ms:     采集 6 路视频帧（并行）
├── T0-20ms:  YOLOv8s 目标检测 (6帧串行) → ~120ms
├── T120ms:   YOLO-Pose 姿态估计 (仅人体区域) → ~60ms
├── T180ms:   行为分类推理 → ~20ms
├── T200ms:   结果聚合与输出
└── 剩余 ~800ms: 等待下一帧

结论：完全可行！
- 单帧总处理时间: ~200ms < 1000ms 帧间隔
- NPU 利用率: ~20% (非满载)
- 内存占用: ~500MB (模型 + 缓冲区)
```

#### 3.1.4 性能优化策略

| 优化手段 | 效果 |
|----------|------|
| **1fps 采样** | 降低 30 倍计算量（从 30fps 降到 1fps） |
| **NPU 加速** | INT8 推理比 CPU 快 10 倍 |
| **模型量化** | INT8 模型体积减少 75% |
| **区域检测** | 只对有人区域进行姿态估计 |
| **批处理** | 共享预处理（可选） |

## 六、总结

### 6.1 新方案核心变更

| 项目 | 原方案 | **新方案** |
|------|--------|-----------|
| AI 框架 | MediaPipe + ONNX | **纯 YOLO + ONNX** |
| 行为识别 | MediaPipe Pose | **YOLO-Pose + 时序分类** |
| 人脸检测 | MediaPipe Face | **YOLO-Face** |
| 视频帧率 | 实时 30fps | **降采样 1fps** |
| 并发路数 | 4 路 | **6 路 1080P** |

### 6.2 可行性确认

✅ **6 路 1080P@1fps 完全可行**
- NPU 算力充足：6TOPS 算力，处理 6 路仅需 20% 利用率
- 1fps 大幅降低负载：从 180fps 降到 6fps，负载降低 30 倍
- 内存充足：8GB RAM，轻松支持多模型并发

✅ **纯 YOLO 方案可行**
- YOLOv8s: 高精度目标检测 (80+ 类)
- YOLO-Pose: 17 关键点姿态估计
- YOLO-Face: 高效人脸检测
- 统一 ONNX Runtime + RKNN NPU 加速
