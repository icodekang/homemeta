# 智能家庭陪伴机器人 - 硬件与软件详细方案

## 项目概述

基于 AI Agent 的智能家庭陪伴机器人，定位为家庭情感陪伴助手 + 智能管家。参考瓦利(WALL-E)的可爱形象，具备情感表达、语音交互、视觉感知、自主移动、智能控制等功能。

---

### 1.2 核心硬件选型

#### 主控平台对比

| 方案 | 芯片 | CPU | NPU | GPU | 内存 | 功耗 | 适合场景 |
|------|------|-----|-----|-----|------|------|----------|
| **推荐** | **RK3588** | 4×A76 + 4×A55 | 6TOPS | Mali-G610 | 8GB | 15W | 旗舰方案 |
| 备选1 | Jetson Nano | 4×A57 | 0.5TOPS | Maxwell | 4GB | 10W | 入门方案 |
| 备选2 | Orin Nano | 6/8核 | 40TOPS | Ampere | 8GB | 15W | 高性能 |

**推荐理由**：RK3588 性价比高，NPU 6TOPS 足够 YOLO 推理，支持 8K 编解码，生态成熟。

### 2.2 AI Agent 详细设计

#### 2.2.1 Agent 架构 (LangGraph)

```python
# Agent 工作流程
from langgraph.graph import StateGraph
from langchain_core.messages import HumanMessage, AIMessage

# 状态定义
class RobotState(TypedDict):
    messages: List[BaseMessage]          # 对话历史
    observation: Dict                   # 感知到的环境
    memory: Dict                        # 长期记忆
    emotion: str                        # 当前情感状态
    intent: str                          # 用户意图
    plan: List[str]                     # 执行计划
    action_result: Any                  # 行动结果
```

#### 2.2.2 Agent 核心能力

| 能力 | 实现方式 | 功能 |
|------|----------|------|
| **多模态感知** | 视觉(Vision) + 语音(ASR) | 理解用户和环境 |
| **意图理解** | LLM + Few-shot | 解析用户意图 |
| **工具调用** | LangChain Tools | 控制设备、查询信息 |
| **对话管理** | LangGraph | 多轮对话、上下文记忆 |
| **情感计算** | 规则 + AI | 生成恰当情感响应 |
| **自主决策** | ReAct/CoT | 主动服务、主动互动 |
| **长期记忆** | 向量数据库 | 记住用户偏好、习惯 |

### 2.3 对话与语音系统

#### 2.3.1 语音交互流程

```
用户说话 → 麦克风阵列 → Whisper (ASR) → LLM (理解) 
                                              ↓
                              Agent 处理 → LLM (生成) → TTS → 扬声器
                                              ↓
                                   动作/表情/控制响应
```

#### 2.3.2 语音配置

| 组件 | 选型 | 说明 |
|------|------|------|
| ASR | Whisper (本地) | 离线语音转文字 |
| TTS | Edge TTS / VALL-E | 自然语音合成 |
| 唤醒词 | 本地唤醒词检测 | "你好机器人" / 自定义 |
| 降噪 | WebRTC VAD | 噪声抑制 |

### 2.4 视觉系统

#### 2.4.1 视觉能力

| 能力 | 模型 | 功能 |
|------|------|------|
| 目标检测 | YOLOv8s | 80+ 类物体检测 |
| 人脸识别 | YOLO-Face + ArcFace | 家庭成员识别 |
| 姿态估计 | YOLO-Pose | 17 关键点 |
| 情绪识别 | 基于表情/姿态 | 开心/难过/惊讶等 |
| 手势识别 | YOLO-Hand | 挥手/点赞/OK |

#### 2.4.2 视觉处理流程

```
摄像头 → 帧采集 → YOLO推理 → 结果解析 → 状态更新 → Agent感知
                              ↓
                       避障/跟随/情感/手势
```

## 四、开发计划

### 4.1 里程碑规划

```
阶段一：硬件原型 (Week 1-4)
├── 机械结构组装
├── 核心板焊接测试
├── 传感器驱动调通
```

## 六、总结

| 维度 | 方案 |
|------|------|
| **硬件** | RK3588 旗舰配置 + 轮式移动 + 显示屏眼睛 |
| **软件** | LangGraph Agent + 本地/云端 LLM |
| **AI** | 纯 YOLO (目标/姿态/手势/情绪) |
| **交互** | 语音对话 + 情感表情 + 动作表达 |
| **特色** | 主动陪伴 + 记忆系统 + 智能家居控制 |

这个方案具备成为"家庭成员"的潜力，而不仅仅是一个工具。
